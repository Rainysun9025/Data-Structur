这次学习中遇到的问题
1. 例子本身不符合 AVL 前提
一开始用来演示插入的树：

Text
        50
       /  \
     30    70
    /
   20
  /
 10
它其实已经不平衡了，

所以它不能作为“AVL 树插入前”的合法例子。

问题本质
AVL 树要求：任意节点左右子树高度差不超过 1
这棵树在插入前就可能已经失衡
所以它更像普通 BST，不是合法 AVL
2. 误以为旋转一定发生在插入位置或整棵树根
后来在 AVL 插入里看到旋转代码：

C
if (balance > 1 && key < root->left->key)
    return rightRotate(root);
容易误解成：

旋转的是整棵树的最上面那个根
或者旋转的是插入进去的那个节点
实际情况
旋转发生在：

从插入点往上回溯时，第一个失衡的祖先节点

也就是说，旋转的是当前递归层的 root，它可能只是某个子树的根。

3. 对代码里的 root 变量理解不清
代码里变量名叫 root，容易让人以为它永远是整棵树的根。

实际上
在递归中：

每一层都有自己的 root
这个 root 代表“当前子树的根”
rightRotate(root) 旋转的是当前这棵子树
解决过程
第一步：发现例子不合法
你指出：

“这棵树不是已经不平衡了吗？”

这一步很关键，说明我们意识到：

AVL 插入示例必须先从合法平衡树开始。

第二步：重新理解 AVL 插入前提
我重新澄清了 AVL 的定义：

插入前树必须已经是 AVL
插入后可能失衡
再通过旋转恢复平衡
这样就把“AVL”与“普通 BST”区分开了。

第三步：定位旋转发生的位置
你继续问：

“那它在哪个地方旋转？”

这促使我们明确：

旋转不是固定在根
也不是固定在插入点
而是在第一个失衡的祖先节点
第四步：结合代码理解递归
看到这段代码后：

C
if (balance > 1 && key < root->left->key)
    return rightRotate(root);
我们进一步确认：

root 是当前递归层的子树根
return rightRotate(root) 是把旋转后的新子树根返回给上一层
这样递归才能把平衡后的树一路传回去
最终收获
对 AVL 树有了更准确的认识
插入前树必须是平衡的
插入后从下往上检查平衡
在第一个失衡节点处旋转
旋转对象是当前递归层的 root
对代码的理解更清楚了
root 不是“永远的全局根”，而是：

当前子树的根

所以 AVL 插入代码中的旋转，本质上是在局部子树上做修复。

一句话总结
这次学习中主要遇到两个问题：

一个是示例树本身不满足 AVL 前提，另一个是误以为旋转发生在整棵树根。

通过重新确认 AVL 的定义、分析递归回溯过程，并结合代码理解 root 的含义，最终明确了：AVL 旋转发生在从下往上第一个失衡的祖先节点上，而且这里的 root 只是当前子树的根
